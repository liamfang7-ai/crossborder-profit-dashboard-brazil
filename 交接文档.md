## 当前目标

这是一个内部使用的 Brazil Marketplace 利润检测后台，用于查看订单、SKU 销售、成本、费用公式和人民币利润。

核心规则：
- 只保留最近 90 天订单和 SKU 明细。
- 90 天以前订单自动删除，不归档、不恢复。
- Dashboard 重点看 SKU 汇总利润。
- 成本和费用集中在 SKU 产品管理维护。
- 订单明细页只读，不编辑成本。
- 备用 CSV 导入保留。
- 不显示退款费用，不把 `refund_amount` 纳入 Dashboard SKU 利润。
- 时间统一使用 `America/Sao_Paulo`。

## 主要页面

- `/`：Dashboard SKU 汇总利润看板。
- `/login`：内部账号密码登录页。
- `/mercadolibre`：third-party marketplace OAuth 连接和手动同步订单。
- `/products`：SKU 产品管理、成本、费用公式、汇率设置。
- `/orders`：订单明细，只读查看，支持日期筛选和搜索。
- `/import`：备用 CSV 导入。
- `/archive`：已停用，仅显示提示。

## SKU 利润公式

```text
sales_cny = sales_brl * exchange_rate
product_cost_cny = products.unit_cost_cny * quantity
shipping_cost_cny = products.unit_shipping_cost_cny * quantity
fees_total_cny = (platform_fee_brl + platform_tax_brl + last_mile_fee_brl + ad_cost_brl + other_fee_brl) * exchange_rate
profit_cny = sales_cny - product_cost_cny - shipping_cost_cny - fees_total_cny
profit_margin = sales_cny > 0 ? profit_cny / sales_cny : 0
```

安全公式文件：`src/lib/safe-formula.ts`

允许变量：
- `sales_brl`
- `quantity`
- `order_count`
- `unit_price_brl`
- `exchange_rate`

禁止：
- `eval`
- `new Function`
- 任意 JavaScript 执行

## 日期筛选

Dashboard 和 `/orders` 都支持：
- 今日
- 本月
- 近7天
- 近30天
- 近90天
- 自定义日期范围

统一时间工具：`src/lib/market-time.ts`

自定义限制：
- 开始日期不能早于最近 90 天。
- 结束日期不能晚于当前巴西日期。
- 开始日期不能大于结束日期。

## 自动清理

接口：`/api/maintenance/cleanup-old-data`
文件：`src/app/api/maintenance/cleanup-old-data/route.ts`

功能：
- 删除 `order_items` 中 90 天前数据。
- 删除 `orders` 中 90 天前数据。
- 不删除 products、tokens、settings。

安全：
- 使用 `CRON_SECRET`
- 请求头：`Authorization: Bearer CRON_SECRET`

Vercel Cron：`vercel.json`

## 最近踩过的坑

1. `dashboard-client.tsx` 曾出现中文乱码，导致 Vercel 构建报 `Unterminated string constant`，已通过重写 UTF-8 文案修复。
2. 登录保护一开始使用 `proxy.ts`，线上首页 `/` 没有正确拦截，已改为 `middleware.ts` 并显式 matcher 覆盖 `/`。
3. `.next/dev/types` 可能残留旧路由生成类型，如果本地 `tsc` 报已删除页面找不到，清理 `.next` 后再跑。
4. 当前分支曾显示为 `feature/product-content-generator`，但它和 `main` 当时指向同一 commit。登录保护建议提交到单独分支，例如 `feature/login-protection`。
5. `product-content-generator` 相关文件不是登录保护必需功能，不应混入登录保护提交。
